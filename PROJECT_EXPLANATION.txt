---------------------------------------------------------------------
WHAT IS THIS PROJECT?
---------------------------------------------------------------------

This is a web-based AI Financial Assistant. Think of it like a smart
money advisor that you can open in your browser. You type in your
financial details (like your income, expenses, or a loan amount) and
the app gives you advice, calculations, and feedback.

It has two main parts:
  - A FRONTEND (what you see in the browser) — built with React.js
  - A BACKEND (the brain behind the scenes) — built with Python


---------------------------------------------------------------------
HOW THE APP WORKS (SIMPLE FLOW)
---------------------------------------------------------------------

1. You open the app in your browser (React frontend)
2. You enter some financial information (e.g., monthly income)
3. The browser sends that info to the Python backend (Flask server)
4. The backend:
      a. Does the math (calculations)
      b. Asks an AI (Groq) to write a helpful tip or insight
      c. Runs that tip through a TRAINED MODEL to judge whether
         the financial situation is Positive, Negative, or Neutral
5. The result comes back to your browser and displays nicely


---------------------------------------------------------------------
WHAT APIS ARE USED?
---------------------------------------------------------------------

APIs are like remote services your app talks to over the internet.

1. GROQ API (most important AI API)
   - Service: Groq (runs a powerful AI called LLaMA 3.3 70B)
   - What it does: Generates financial advice and tips in plain English
   - Example: When you enter your budget, Groq writes something like
     "Your savings rate is good, but try to cut down on dining out."
   - It is used in almost every feature of the app
   - For the chat feature, Groq streams the reply word by word
     (like ChatGPT typing in real time)

2. OPEN EXCHANGE RATES API
   - Service: Free online currency data
   - What it does: Gets live exchange rates (USD, PKR, EUR, etc.)
   - Used in the Currency Converter feature
   - No login/key required (uses free public endpoint)

These two APIs are the only external services the app depends on.


---------------------------------------------------------------------
THE TRAINED MODEL — THE MOST IMPORTANT PART
---------------------------------------------------------------------

The app has its own trained machine learning model. This is separate
from the Groq AI. This model was trained by us, lives on our computer,
and does one specific job: SENTIMENT ANALYSIS.


WHAT IS SENTIMENT ANALYSIS?
----------------------------
Sentiment analysis means reading a piece of text and deciding whether
it sounds Positive, Negative, or Neutral.

  Example:
  - "Your savings are excellent and growing fast" → POSITIVE (Green)
  - "You are spending more than you earn, danger ahead" → NEGATIVE (Red)
  - "Your loan amount is 500,000 rupees" → NEUTRAL (Yellow)


WHAT KIND OF MODEL IS IT?
--------------------------
Model Type: SGDClassifier (Stochastic Gradient Descent Classifier)
  - This is a text classification model from a Python library
    called scikit-learn
  - It reads text and learns to sort it into categories
  - It is lightweight and fast (does not need a GPU)

Text Vectorizer: HashingVectorizer
  - Before the model can read text, text must be converted to numbers
  - HashingVectorizer breaks text into words and word-pairs (called
    n-grams) and converts them to a giant list of numbers
  - Example: "profit increased" → [0, 0, 1, 0, 0, 1, 0, ...]
  - It handles 131,072 possible features (word combinations)

Output of the model for each text:
  - Label: positive / negative / neutral
  - Emoji: Green circle / Red circle / Yellow circle
  - Confidence: e.g., 87% (how sure the model is)
  - Probabilities: exact score for all 3 categories


---------------------------------------------------------------------
WHERE DID THE TRAINING DATA COME FROM?
---------------------------------------------------------------------

The model learned from 3 sources of data (called training data):


SOURCE 1 — HANDCRAFTED PHRASES (written by us)
-----------------------------------------------
We manually wrote 400+ financial sentences with their correct labels.

  Positive examples we wrote:
  - "Strong revenue growth reported this quarter"
  - "Excellent savings rate, you are on track"
  - "Your investment returns are outstanding"

  Negative examples we wrote:
  - "Bankruptcy risk is increasing rapidly"
  - "Your debt is dangerously high"
  - "Revenue declined sharply this year"

  Neutral examples we wrote:
  - "The loan interest rate is 12 percent"
  - "Your monthly expenses total 30,000 rupees"
  - "Tax filing deadline is approaching"

These are all stored directly in the Python file: sentiment_model.py


SOURCE 2 — FINANCIAL PHRASEBANK (external dataset)
---------------------------------------------------
This is a well-known, publicly available dataset made by researchers
and financial experts. It contains 2,264 real financial sentences that
were labeled by human experts.

  Where it comes from: GitHub (downloaded automatically during training)
  Who made it: Academic researchers in financial NLP
  What it contains: Real sentences from financial news and reports,
                    each labeled as positive, negative, or neutral

This dataset makes our model much better because it has seen real
financial language used in the real world, not just phrases we wrote.

Combined with our handcrafted data, total training data = ~2,600+ sentences.


SOURCE 3 — LIVE APP DATA (grows over time)
-------------------------------------------
Every time a user uses a feature (Budget, Savings, Loan, etc.):
  - The Groq AI generates a tip/insight
  - That text is automatically saved to our database (database.db)
  - It is also given a label (positive/negative/neutral) based on
    the user's actual financial numbers

  Example logic:
  - If savings rate >= 20%  → label the insight as "positive"
  - If savings rate < 5%    → label as "negative"
  - Otherwise               → label as "neutral"

This means the model's training data keeps growing the more people
use the app. We can retrain the model at any time using this data.


---------------------------------------------------------------------
HOW WAS THE MODEL TRAINED?
---------------------------------------------------------------------

Training means teaching the model by showing it examples.

STEP-BY-STEP TRAINING PROCESS:
1. Load all 400+ handcrafted phrases from our Python file
2. Download the 2,264 Financial PhraseBank sentences from the internet
3. Load any additional examples saved in our database
4. Convert all text into numbers using HashingVectorizer
5. Split data: 85% used for training, 15% used for testing
6. Run SGDClassifier.fit() — this is the actual learning step
7. Test accuracy on the 15% test data
8. Save the trained model to a file: financial_sentiment_model.pkl

The .pkl file is like a save file for the model. Once trained, the
model is loaded from this file every time the app starts — it does
not retrain from scratch every time.

Model file size: 3.1 MB
Location: backend/financial_sentiment_model.pkl


INCREMENTAL LEARNING (learning on the go):
------------------------------------------
The model can also learn new examples one at a time without full
retraining. This is called partial_fit (incremental learning).

After every user interaction, the new text + label can be fed to
the model to slightly update its knowledge.

Full retraining can also be triggered via a special API endpoint
(/api/retrain) which retrains the model using everything in the
database at that moment.


---------------------------------------------------------------------
HOW THE TRAINED MODEL IS USED IN FEATURES
---------------------------------------------------------------------

The model is used in 6+ features. Here is what happens in each:


BUDGET BREAKDOWN FEATURE:
  - User enters: income, expenses, savings amount
  - Groq generates: a budget advice tip
  - Model classifies: is this situation Positive, Negative, Neutral?
  - Logic: savings rate >= 20% = Positive, < 5% = Negative
  - Display: green/red/yellow badge shown next to the advice


SAVINGS GOAL PLANNER:
  - User enters: savings target and current savings
  - Groq generates: a motivational savings tip
  - Model classifies: Positive if >50% progress made, Negative if goal
    seems very hard to reach
  - Display: badge shown with confidence percentage


LOAN / EMI CALCULATOR:
  - User enters: loan amount, interest rate, tenure
  - Groq generates: advice on managing the loan
  - Model classifies: Positive if interest ratio < 20% of income,
    Negative if > 50% of income goes to loan
  - Display: shows how healthy or dangerous the loan situation is


INVESTMENT CALCULATOR:
  - User enters: amount invested, duration, expected return
  - Groq generates: investment strategy insight
  - Model classifies: Positive if ROI >= 30%, Negative if ROI < 0%
  - Display: shows investment sentiment badge


NET WORTH TRACKER:
  - User enters: assets and liabilities
  - Groq generates: overall financial health assessment
  - Model classifies: Positive if net worth > 0, Negative if < 0
  - Display: shows overall financial health badge


TAX ESTIMATOR (Pakistan FBR 2026):
  - User enters: annual income
  - Groq generates: tax-saving tips
  - Model classifies: Positive if effective tax rate < 10%,
    Negative if > 25%
  - Display: shows tax burden sentiment


SENTIMENT ANALYZER (standalone feature):
  - User types: any financial text directly
  - No Groq involved — model runs directly on user's text
  - Display: full breakdown with label, confidence %, and all
    3 class probabilities shown in a visual chart


---------------------------------------------------------------------
THE DATABASE
---------------------------------------------------------------------

The app uses a simple local database called SQLite.
Database file: backend/database.db

It has 3 tables (like spreadsheets):

  1. users — stores username and password (password is hashed/encrypted)
  2. history — stores past calculations made by each user
             (last 20 saved per user)
  3. sentiment_training_data — stores AI-generated insights + labels
             (this grows over time and is used to retrain the model)

---------------------------------------------------------------------
WHAT MAKES THIS PROJECT SPECIAL (KEY HIGHLIGHTS)
---------------------------------------------------------------------

1. OWN TRAINED MODEL
   We did not just use a pre-built AI. We trained our own sentiment
   classification model from scratch using real financial data.

2. TWO TYPES OF AI WORKING TOGETHER
   - Groq (LLaMA) generates the advice in natural language
   - Our trained model judges/classifies that advice
   Both are used at the same time for every feature.

3. GROWING DATASET
   Every user interaction adds new training data to the database.
   The model can be retrained to improve over time.

4. INCREMENTAL LEARNING
   The model supports partial_fit, meaning it can learn from new
   examples one at a time without forgetting what it already knows.

5. FULL STACK PROJECT
   It covers everything: trained ML model, REST API backend,
   live database, and a complete multi-page React frontend.

6. REAL-WORLD DATA
   We used the Financial PhraseBank, a real dataset used in academic
   NLP research, to make our model understand genuine financial language.
